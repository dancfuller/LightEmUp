# LightEmUp

A local-network web app for controlling Philips Hue and Govee smart lights from a single interface. No cloud, no accounts — everything runs on your LAN.

![Python](https://img.shields.io/badge/Python-3.10+-blue) ![FastAPI](https://img.shields.io/badge/FastAPI-latest-009688) ![License](https://img.shields.io/badge/License-MIT-green)

## What It Does

LightEmUp discovers and controls Philips Hue (Zigbee via bridge) and Govee (LAN/UDP) smart lights through one unified web UI. Group lights into rooms, set colors and brightness, save favorites, and run scene effects — all without leaving your local network.

### Key Features

- **Unified control** — Hue and Govee devices in one UI with room grouping
- **Per-light controls** — on/off, brightness, color picker (Hue Bar or Color Wheel — your pick in Settings), RGB sliders, favorite colors. Brightness slider can be adjusted while the light is off.
- **Per-light Whole / Segments mode toggle** — for segment-capable Govee devices, choose between single-color control and per-panel control with one toggle (persisted per device). Segments mode shows tappable swatches with the picker bound to the selected segment.
- **Room-level overrides** — on/off, brightness, and color for all lights in a room at once
- **Interactive room maps** — 2D floor plan and linear layout modes, drag-and-drop placement, live device state, fixture outlines, reference furniture items, and auto-save. The map lives as a collapsible section inside Room Controls.
- **Lightning storm scene** — realistic thunderstorm simulation with Hue flashes, configurable timing presets, configurable storm start delay (0–120s), optional thunder audio, and a "funny mode" that replaces thunder with 20 fart sound effects. Fixture members flash in unison as a single light source.
- **Color Mode** — room-level color schemes with layout awareness: **Palette** (curated 8-color sets with adjacency repair), **Gradient** (shades of one color along a direction), **Tonal** (random shades of one color, no adjacent duplicates), **Custom** (1–4 user-chosen seed colors with a "Create shades" or "Use exact seed colors" toggle), and **Beacon** (one color radiating from a source with falloff). 16+ preset palettes.
- **Address Segments toggle** — room scenes can address segmented devices either **individually** (default — each panel gets its own color) or **as a unit** (whole-device single color). Switching syncs every device's per-light mode automatically.
- **Fixtures** — group multiple bulbs that share a physical housing (e.g. a triple-bulb sconce) so palette modes never repeat colors within the housing and lightning flashes all members in unison.
- **Strict palette adjacency** — palette generation uses a multi-pass algorithm: forward greedy assignment, weighted local-search swap, and a final fixture/segment-sibling repair so fixture mates and hexa panels always end up with distinct colors.
- **Per-segment color control** — independent color control of individual panels/spotlights. H6061 Hexa Panels use the Razer LAN protocol; H7065/H7066 Outdoor Spotlights use the Govee Platform API v2. State is tracked server-side and survives page reloads. The Razer keepalive auto-refreshes hexa colors every 45 s so static palettes don't revert.
- **Segment-preserving brightness** — moving the brightness slider on a Govee device in Segments mode re-scales every segment's color rather than knocking the device out of segment mode.
- **Minimum saturation floor** — toggle in Settings (default on, 35%) that boosts any color generated by a scene mode below the threshold up to the threshold, keeping washed-out near-white shades out of generated palettes/gradients/etc.
- **Missing device tracking** — every Govee device the server has ever seen is remembered. If a device doesn't answer a re-scan it surfaces in Settings as greyed-out with a red dot, an inline re-scan button, and an × to forget it permanently.
- **Layout Identify mode** — one-click button assigns each light/segment a unique color based on physical proximity, making it easy to match map nodes to real-world lights
- **Server logs viewer** — Settings → Server → View Logs shows the last N lines of a 48-hour rotating log file with level filter, search, auto-refresh tail, and copy.
- **Versioning** — semantic version + git hash + commit date shown in Settings → About so the deployed build is always identifiable.
- **Device nicknames** — custom names persisted across sessions
- **Server management** — restart or shut down from the Settings tab; supports both Windows (standalone) and Linux (systemd)
- **280+ Govee SKU names** — automatic product name resolution
- **Mobile-friendly** — responsive design throughout
- **Mostly local** — Hue, Govee LAN control, and H6061 hexa segments are fully local. Per-segment control of H7065/H7066 Outdoor Spotlights uses the Govee Platform API v2 (cloud) — disable that path or skip those devices if you need true air-gapped operation. Frontend CDN scripts (React/Babel) are cached after the first load.

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/dancfuller/LightEmUp.git
cd LightEmUp/backend
python -m venv venv
```

**Windows (PowerShell):**
```powershell
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

**Linux/macOS:**
```bash
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Start the server

```bash
python main.py
```

Open `http://localhost:8420` in your browser. That's it.

### 3. Connect your lights

- **Hue**: Click the Hue badge in the header, press the physical button on your bridge, then click Pair
- **Govee**: Devices are auto-discovered if LAN Control is enabled in the Govee Home app

### Desktop Shortcut (Windows)

Run the shortcut installer for a one-click launcher on your Desktop and Start Menu:

```powershell
powershell -ExecutionPolicy Bypass -File install-shortcut.ps1
```

This creates a shortcut that silently starts the server (if not already running) and opens the browser — no console window.

## Lightning Storm Scene

A scene engine that simulates realistic thunderstorms by flashing lights in randomized bursts with synced thunder audio.

| Preset      | Gap Between Flashes | Burst Size | Feel                    |
|-------------|---------------------|------------|-------------------------|
| Realistic   | 15–60 seconds       | 1–2        | Distant summer storm    |
| Slow        | 5–20 seconds        | 1–2        | Approaching storm       |
| Medium      | 1.5–6 seconds       | 1–3        | Active storm            |
| Fast        | 0.5–2 seconds       | 2–5        | Intense storm           |
| Epilepsy    | 80–400 ms           | 4–8        | Strobe (use with care)  |

Features:
- **Thunder sound effects** — browser-synthesized thunder synced to flashes via Server-Sent Events
- **Funny mode** — replaces thunder with 20 real fart sound effects
- **Immediate vs. delayed** thunder timing toggle
- **Storm start delay** — configurable 0–120 second delay (default 7 s). Lights drop to the dim background immediately on start, then hold for the delay before any flashing or thunder begins.
- **Govee flash participation** — Govee lights can flash alongside Hue, or hold a background glow. (Note: Govee LAN devices animate color changes too slowly to produce a crisp flash; the toggle defaults to background-only.)
- **Per-segment flashing** — on compatible Govee devices (e.g. Hexa Panels), individual panels flash independently
- **Fixture-aware flashing** — bulbs grouped into a fixture share a single flash pattern so the whole housing reads as one light source. Multi-segment Govee devices in a fixture also collapse to whole-device mode.
- **Advanced controls** — fine-tune gap timing, flash duration, burst count, and inter-burst gaps

## Architecture

```
backend/
  main.py             # FastAPI server — endpoints, config persistence
  discovery.py        # Device discovery & control (Hue REST, Govee UDP, Razer, V2 cloud)
  scenes.py           # Lightning scene engine with async pattern generation
  razer_keeper.py     # Background refresher that re-sends razer segment colors
                      # every 45s so hexa panels don't revert to default
  segment_state.py    # In-memory tracker of last-known per-segment colors
                      # + device-level brightness, exposed via /api/govee/segment-state
  version.py          # Single source of truth for the app version
  requirements.txt    # Python deps (fastapi, uvicorn, httpx)
  start.ps1           # PowerShell launcher
  config.json         # LOCAL ONLY (gitignored) — IPs, credentials, rooms, prefs
  logs/               # Rotating server logs (48h retention, gitignored)
  static/
    index.html        # HTML shell — loads React/Babel CDN + component scripts
    js/
      utils.js              # React hooks, API wrapper, color utilities, SKU names,
                            # PickerStyleContext (HueBar / ColorWheel)
      audio.js              # Thunder synth, fart sounds
      components-shared.js  # ColorPicker, ColorWheel, HueBar, Slider, RgbSliderInput
      light-card.js         # Per-device card (Whole/Segments toggle, segment swatches)
      lightning-panel.js    # Lightning storm scene UI
      room-map.js           # 2D floor plan & linear layout (drag/drop, fixtures)
      color-mode.js         # Room scenes: Palette, Gradient, Tonal, Custom, Beacon
      room-section.js       # Room grouping; Map lives inside Room Controls
      room-assignment.js    # Drag-based room assignment UI
      setup-wizard.js       # Hue Bridge pairing wizard
      server-logs.js        # Server logs modal
      app.js                # Main App component + render
    sounds/             # Audio assets (fart sound effects for funny mode)
deploy/                 # Raspberry Pi deployment artifacts (see below)
  install.sh            # First-time installer (apt deps, venv, systemd, avahi)
  update.sh             # Pull-and-restart deploy helper (run via SSH)
  lightemup.service     # systemd unit (User=pi, Restart=always)
  lightemup.avahi.service  # mDNS advertisement so lightemup.local resolves
  README.md             # Pi setup walkthrough
launch.ps1              # Silent launcher (starts server + opens browser)
launch.vbs              # VBScript wrapper for no-console-window launch
install-shortcut.ps1    # Desktop/Start Menu shortcut installer
```

- **Backend**: Python FastAPI. Hue via local REST API, Govee via UDP (LAN protocol + Razer per-segment) and httpx (V2 cloud per-segment).
- **Frontend**: Modular React app split across 12 component files — React and Babel load from CDN, no build step, Babel transpiles in-browser. A single `PickerStyleContext` is the only context; everything else flows as props.
- **Config**: `config.json` stores bridge credentials, rooms, nicknames, room layouts, fixtures, lightning settings, per-device modes, known Govee devices, and UI prefs (gitignored).
- **Logging**: root logger writes to `backend/logs/server.log` with `TimedRotatingFileHandler` rotating hourly, 48 backups retained. Console output is preserved for `journalctl`.

## Govee Per-Segment Control

Two protocols are used for per-segment color control depending on the device:

### Razer Protocol (LAN, H6061 Hexa Panels)
An undocumented binary protocol sent over the local UDP port. The full N-segment color array is sent in one packet. Razer mode auto-disables after ~60 seconds of no LED data, so the backend's `razer_keeper` re-sends the same packet every 45 seconds to keep static colors visible indefinitely. The keepalive is cancelled automatically when the user issues a whole-device command or starts a lightning scene on the device.

The same protocol is used by the lightning scene to flash individual Hexa panels independently — that path uses its own keepalive plus per-segment patterns.

**Confirmed working:** H6061 Glide Hexa Light Panels (7 segments)

### Govee Platform API v2 (Cloud, H7065/H7066 Outdoor Spotlights)
The Outdoor Spotlights 2-Pack and 4-Pack do not respond to the Razer protocol. Per-segment control is only available via the Govee Platform API v2. A Govee Developer API key is required — set `govee_api_key` in `config.json`.

**Confirmed working:** H7065 Outdoor Spotlights 2-Pack (2 segments), H7066 4-Pack (4 segments)

The V2 API enforces a ~1 req/sec rate limit, so applying a color gradient to multiple segments takes a few seconds.

### How to use it from the UI

- **Per light**: each segment-capable Govee device in the Lights list has a "Whole light / Segments" toggle (persisted per device, accessible while the light is off). In Segments mode, tap a segment swatch to select it, then pick a color — the brightness slider re-scales every segment so colors are preserved.
- **At the room level**: open Color Mode and use the "Address segments" toggle at the top. **Individually** (default) treats each panel as its own entry for Palette/Gradient/Tonal/Beacon/Custom; **As a unit** treats the device as a single light.
- **On the map**: each segment-capable device has an inline expand badge to lay out its panels explicitly on the floor plan; segment positions are used by Gradient and Beacon for spatial variation.

## Raspberry Pi Deployment

The `deploy/` directory contains everything needed to run LightEmUp 24/7 on a Raspberry Pi (tested on Pi 4B with Pi OS Lite 64-bit). After flashing the SD card and SSHing in:

```bash
git clone https://github.com/dancfuller/LightEmUp.git ~/lightemup
bash ~/lightemup/deploy/install.sh
```

`install.sh` installs apt deps, creates the venv, installs `lightemup.service` under systemd with `Restart=always`, and registers an Avahi mDNS advertisement so the Pi resolves as `lightemup.local` on the LAN.

Subsequent updates from a dev machine:

```bash
ssh -t pi@lightemup '~/lightemup/deploy/update.sh'
```

`update.sh` pulls main, refreshes pip deps, reinstalls the systemd unit if it changed, and restarts the service.

See `deploy/README.md` for the full walkthrough.

## Firewall

If accessing from another device on your network, allow the server port:

```powershell
New-NetFirewallRule -DisplayName "LightEmUp" -Direction Inbound -Protocol TCP -LocalPort 8420 -Action Allow
```

If Govee devices aren't discovered, allow the UDP discovery ports:

```powershell
New-NetFirewallRule -DisplayName "Govee LAN Discovery" -Direction Inbound -Protocol UDP -LocalPort 4001,4002 -Action Allow
```

## Requirements

- **Python 3.10+** — must be on your system PATH
- **Internet connection** — required on first launch only (React/Babel load from CDN)
- Philips Hue Bridge on the same network (optional)
- Govee devices with LAN Control enabled in the Govee Home app (optional)

### Windows Notes

- **PowerShell execution policy**: If you get "running scripts is disabled", run once in an admin PowerShell:
  ```powershell
  Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```
- **Python on PATH**: When installing from [python.org](https://www.python.org/downloads/), check "Add python.exe to PATH"

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `python` not recognized | Install Python and ensure it's on PATH. Restart your terminal. |
| Scripts disabled on this system | Run `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` in admin PowerShell |
| UI is blank | Frontend needs internet on first load (CDN). Check your connection and refresh. |
| Port 8420 in use | Check with `netstat -ano \| findstr :8420` and close the conflicting process |
| Govee devices not found | Enable LAN Control in Govee Home app. Allow UDP 4001/4002 through firewall. |
| Can't access UI from phone | Allow TCP 8420 through firewall (see [Firewall](#firewall)) |

## Known Limitations

- Govee Razer per-segment control only confirmed on H6061 Hexa Panels (other razer SKUs may work but are untested)
- Govee LAN commands are fire-and-forget — the app uses optimistic UI updates
- Razer protocol has a ~60-second timeout — handled by the `razer_keeper` (static colors) and the scene engine (lightning), but if the backend restarts, the hexa panels revert until the next apply
- Govee LAN devices animate color changes too slowly to produce a crisp lightning flash — the `govee_flash` toggle in lightning settings defaults off for that reason
- Cloud V2 per-segment control is rate-limited (~1 req/sec); applying a palette to a 7-panel device takes several seconds
- Unknown Govee SKUs fall back to showing the model number
